<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JavaScript → Rust | JS code calls crypto.subtle methods; all key material handled in Rust |
| Rust → ring | Rust passes key material to ring crate for crypto operations |
| Memory → External | Key material in heap must be zeroed when key dropped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | CryptoKey heap memory | mitigate | Implement zeroization on Drop for key material (use zeroize crate) |
| T-09-02 | Denial of Service | Key generation CPU usage | accept | Key gen is inherently expensive; rate limit at higher layer if needed |
| T-09-03 | Tampering | Key extraction from non-extractable keys | mitigate | Honor extractable flag strictly; never export non-extractable keys |
| T-09-04 | Spoofing | Algorithm confusion attacks | mitigate | Validate algorithm name strictly against allowlist before operations |
| T-09-05 | Elevation | Weak key generation | mitigate | Use ring's secure random for all key generation, never custom CSPRNG |
</threat_model>
