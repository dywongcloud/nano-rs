# Phase 14 Plan 03: VFS State Capture

## Objective
Implement VFS state capture to collect all files from a running isolate's virtual filesystem and prepare them for inclusion in the sliver archive.

## Context
- VFS module exists at `src/vfs/` with MemoryBackend and other backends
- Phase 11-12 implemented VFS with per-isolate namespaces
- Sliver format includes vfs/ prefix for filesystem entries (Phase 13)
- MemoryBackend has snapshot methods from Phase 13

## Tasks

### Task 1: Create VFS Capture Module
**Type:** auto
**Description:** Create `src/sliver/vfs_capture.rs` for VFS state extraction

Implement VFS capture functionality:
- `VfsCapture` struct that walks the VFS
- `capture_from_isolate(isolate)` method to extract all files
- Support for different VFS backends (memory, disk)
- Returns list of (path, content) pairs

**For MemoryBackend:**
- Use existing `entries()` method to get all files
- Extract content from each entry

**For DiskBackend:**
- Walk the on-disk storage directory
- Read files into memory

**Done when:**
- [ ] `src/sliver/vfs_capture.rs` exists with VfsCapture implementation
- [ ] Can capture all files from an isolate's VFS namespace

### Task 2: Handle Directory Walking
**Type:** auto
**Description:** Implement recursive directory traversal for VFS

Add support for:
- Walking directory trees recursively
- Collecting file metadata (size, modified time if available)
- Handling symlinks (or rejecting them for security)
- Filtering out special files

**Done when:**
- [ ] Recursive walking works for nested directories
- [ ] All files in VFS are captured

### Task 3: Add Tests for VFS Capture
**Type:** auto
**Description:** Create tests for VFS state capture

Add tests:
- Test capturing empty VFS
- Test capturing VFS with single file
- Test capturing nested directory structure
- Test capturing binary files
- Test that captured data matches original

**Done when:**
- [ ] All tests pass
- [ ] Test coverage includes edge cases
