# Phase 14 Plan 04: Sliver Creation Integration

## Objective
Integrate all components (CLI, V8 snapshot, VFS capture) into a complete sliver creation pipeline. This is the end-to-end integration that produces .sliver files from running apps.

## Context
- CLI commands defined in 14-01
- V8 snapshot creation in 14-02
- VFS capture in 14-03
- Sliver packer from Phase 13 at `src/sliver/packer.rs`
- Metadata module from Phase 13 at `src/sliver/metadata.rs`

## Tasks

### Task 1: Implement Sliver Create Handler
**Type:** auto
**Description:** Wire CLI `sliver create` to full creation pipeline

Implement the complete sliver creation flow:
1. Parse CLI arguments (hostname, output, name, tag)
2. Find the app/isolate by hostname
3. Create V8 snapshot (heap.bin)
4. Capture VFS state
5. Generate metadata (hostname, created_at, version, etc.)
6. Use SliverPacker to create tar archive
7. Write to output file

**CLI Arguments:**
```
nano-rs sliver create <hostname> --output <file.sliver>
nano-rs sliver create <hostname> --name <name> --tag <tag>
```

**Done when:**
- [ ] `sliver create` command executes full pipeline
- [ ] Creates valid .sliver file with all components
- [ ] Error handling for missing hostname, failed snapshot, etc.

### Task 2: Implement Sliver List Handler
**Type:** auto
**Description:** Implement `sliver list` command

List all slivers in the sliver store:
- Read metadata from each .sliver file
- Display in tabular format (name, hostname, tag, created_at)
- Handle cases where sliver store is empty

**Done when:**
- [ ] `sliver list` displays all slivers
- [ ] Table format is readable

### Task 3: Implement Sliver Delete Handler
**Type:** auto
**Description:** Implement `sliver delete` command

Delete sliver by name:
- Find sliver file by name
- Confirm deletion (or use --force flag)
- Remove file from sliver store
- Handle "not found" gracefully

**Done when:**
- [ ] `sliver delete <name>` removes the sliver
- [ ] Proper error handling for non-existent slivers

### Task 4: Add Metadata Generation
**Type:** auto
**Description:** Create metadata for sliver

Generate metadata including:
- hostname: The app's hostname
- name: User-provided name or default
- tag: User-provided tag
- created_at: ISO 8601 timestamp
- version: NANO version
- vfs_backend_type: Type of VFS backend used

**Done when:**
- [ ] Metadata is generated and included in sliver
- [ ] All required fields are present

### Task 5: Create Integration Tests
**Type:** auto
**Description:** Add end-to-end integration tests

Create comprehensive tests:
- Test full sliver creation from a running app
- Test sliver list shows created sliver
- Test sliver delete removes sliver
- Test roundtrip: create → verify contents → delete
- Test error cases (invalid hostname, missing output path)

**Done when:**
- [ ] All integration tests pass
- [ ] Tests verify heap.bin is present and valid
- [ ] Tests verify VFS files are captured
- [ ] Tests verify metadata accuracy
