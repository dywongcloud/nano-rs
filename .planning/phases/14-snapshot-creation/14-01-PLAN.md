# Phase 14 Plan 01: CLI Sliver Commands

## Objective
Create the CLI subcommand structure for `nano-rs sliver` with create, list, and delete commands. This provides the user-facing interface for snapshot management.

## Context
- Phase 13 completed the sliver format (tar-based) with packer/unpacker
- The CLI needs to integrate with the existing sliver module at `src/sliver/`
- Current CLI structure is in `src/main.rs` using clap

## Tasks

### Task 1: Create CLI Sliver Module
**Type:** auto
**Description:** Create `src/cli/sliver.rs` with sliver subcommand definitions

Create the sliver CLI module with:
- `SliverCommand` enum with Create, List, Delete variants
- `SliverCreateArgs` struct with hostname, output, name, tag options
- `SliverListArgs` and `SliverDeleteArgs` structs
- Integration with existing CLI structure

**Done when:**
- [ ] `src/cli/sliver.rs` exists with complete command definitions
- [ ] All arguments properly typed with clap derive macros
- [ ] Help text describes each command clearly

### Task 2: Wire CLI Commands to Main
**Type:** auto
**Description:** Add sliver subcommand handling to `src/main.rs`

Integrate the sliver CLI module:
- Add sliver to the main CLI enum
- Wire sliver commands through to handlers
- Create stub handlers that print "not yet implemented"

**Done when:**
- [ ] `nano-rs sliver --help` shows the subcommands
- [ ] All sliver commands are recognized by the CLI parser

### Task 3: Add CLI Tests
**Type:** auto
**Description:** Create unit tests for CLI argument parsing

Add tests in `src/cli/sliver.rs`:
- Test parsing `sliver create api.example.com --output ./api.sliver`
- Test parsing `sliver create api.example.com --name api-prod --tag v1.0`
- Test parsing `sliver list`
- Test parsing `sliver delete api-prod`
- Test invalid argument combinations

**Done when:**
- [ ] All CLI parsing tests pass
- [ ] Test coverage for edge cases (missing required args, invalid formats)
