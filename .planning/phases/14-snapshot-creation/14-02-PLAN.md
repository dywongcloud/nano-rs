# Phase 14 Plan 02: V8 SnapshotCreator Integration

## Objective
Implement V8 SnapshotCreator wrapper to capture the JavaScript heap state from running isolates. This enables serialization of the V8 runtime state into a binary blob.

## Context
- rusty_v8 exposes `v8::SnapshotCreator` API
- The snapshot data is completely opaque to NANO — we just capture and store it
- Must handle external references carefully (don't try to serialize Rust pointers)
- Phase 13 established the sliver format with heap.bin as the V8 heap storage

## Tasks

### Task 1: Create V8 Snapshot Module
**Type:** auto
**Description:** Create `src/v8/snapshot.rs` with SnapshotCreator wrapper

Implement the snapshot creation functionality:
- `SnapshotBuilder` struct wrapping v8::SnapshotCreator
- `create_snapshot(isolate)` function that captures heap state
- Handle external references by registering them with the creator
- Serialize the snapshot blob to a Vec<u8>

**Key considerations:**
- Use `v8::SnapshotCreator::new()` with external references array
- Call `creator.set_default_context(context)` to set the context
- Call `creator.create_blob()` to get the snapshot data
- External references must be registered before creating blob

**Done when:**
- [ ] `src/v8/snapshot.rs` exists with SnapshotBuilder implementation
- [ ] Can create snapshot from an isolate and get binary data
- [ ] External references are properly handled (not serialized)

### Task 2: Integrate with Existing Isolate
**Type:** auto
**Description:** Add snapshot method to Isolate or JsRuntime

Extend the existing isolate/V8 integration:

**Done when:**
- [ ] Isolate can create a snapshot of its current heap state
