**Type:** auto

**Read First:**

**Action:**
Create `src/cli/error.rs` with human-readable error messages:

```rust
pub enum CliError {
    SliverNotFound { name: String, searched_paths: Vec<PathBuf> },
    CorruptedSliver { path: PathBuf, reason: String },
    VersionMismatch { sliver: String, expected: u32, found: u32 },
    SnapshotRestoreFailed { sliver: String, fallback: bool },
    InvalidHostname { input: String, suggestion: Option<String> },
    ConfigError { path: PathBuf, details: String },
    IoError { operation: String, path: PathBuf, source: io::Error },
}

impl fmt::Display for CliError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            CliError::SliverNotFound { name, searched_paths } => {
                writeln!(f, "✗ Sliver '{}' not found", name)?;
                writeln!(f)?;
                writeln!(f, "Searched in:")?;
                for path in searched_paths {
                    writeln!(f, "  • {}", path.display())?;
                }
                writeln!(f)?;
                write!(f, "Create it with: nano-rs sliver create <hostname> --name {}", name)
            }
            // ... other variants
        }
    }
}
```

**Acceptance Criteria:**
- All error types have human-readable Display impl
- Error messages include suggestions for fixes
- Context is provided (paths searched, values expected)
- Proper error chaining with source() for debugging
