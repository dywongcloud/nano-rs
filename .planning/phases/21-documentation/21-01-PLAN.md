- This is **misleading** - 267µs is request dispatch latency AFTER server is running

Add new section before "Security Model" (around line 113):

```markdown
## Per-Worker State Isolation

**Critical architectural property:** Each worker thread maintains completely isolated state.

### How It Works
- One isolate per worker thread (thread-local, never shared)
- Context reset between requests clears JavaScript state
- No shared memory between workers
- Each worker processes requests sequentially for its assigned hostname

### Implications
1. **No shared state** - Global variables don't persist between requests
2. **No shared state across workers** - Worker A cannot see Worker B's data
3. **By design, not a bug** - This is intentional security isolation
4. **Sequential per worker** - Same hostname routes to same worker for cache locality

### State Persistence Options
If you need shared state:
- **External database** - PostgreSQL, Redis, etc.
- **External cache** - Redis, Memcached
- **Future: KV/Durable Objects** - Planned for v2.0 (see ROADMAP.md)

### Why This Design?
- **Security** - Prevents cross-tenant data leakage
- **Predictability** - Each request starts clean, no residue from prior requests
- **Fault isolation** - One worker's crash doesn't affect others
```

Update the "Context Reset" explanation to reference this section.
