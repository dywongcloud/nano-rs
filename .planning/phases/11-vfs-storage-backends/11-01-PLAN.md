# Phase 11 Plan 01: VFS Storage Backends

## Objective

Implement pluggable storage backends for the VFS: DiskBackend for local filesystem persistence and S3Backend for object storage. Enable per-app backend selection via configuration.

## Context

Phase 10 established:
- `VfsBackend` trait (async, object-safe)
- `MemoryBackend` as default in-memory storage
- `IsolateVfs` with namespace isolation
- Namespace format: "hostname::uuid::/path"

This plan extends the VFS with persistent storage options while maintaining the same security and isolation guarantees.

## Success Criteria

1. ✅ DiskBackend persists VFS to local filesystem with directory structure per namespace
2. ✅ S3Backend stores files in S3-compatible object storage
3. ✅ Backend selection is configurable per-app in JSON config
4. ✅ All backends enforce namespace isolation
5. ✅ Unit and integration tests pass for all backends

### Task 2: Create DiskBackend Module
**Type:** auto
**TDD:** true

Create `src/vfs/disk.rs` implementing filesystem persistence:

1. **DiskBackend struct:**
   - `base_path: PathBuf` - root directory for all storage
   - `limits: ResourceLimits` - enforce quotas

2. **Namespace directory mapping:**
   - Sanitize namespace (replace `::` with `_`, special chars)
   - Create subdirectory per namespace: `{base}/{sanitized_namespace}/`
   - Mirror VFS path structure within namespace directory

3. **Atomic write implementation:**
   - Write to temp file: `{path}.tmp.{random}`
   - fsync temp file
   - Atomic rename to final path
   - Clean up temp on error

4. **Implement VfsBackend trait:**
   - `read`: Read file content
   - `write`: Atomic write with quota checks
   - `exists`: Check file existence
   - `delete`: Remove file, update counters
   - `metadata`: Get file stats

**Done Criteria:**
- DiskBackend implements VfsBackend
- Atomic writes work correctly
- Unit tests pass (5+ tests)
- Path traversal prevented

**Files:**
- src/vfs/disk.rs (new)

### Task 3: Create S3Backend Module
**Type:** auto
**TDD:** true

Create `src/vfs/s3.rs` implementing S3-compatible storage:

1. **S3Backend struct:**
   - `bucket: Bucket` from rust-s3
   - `limits: ResourceLimits`
   - `prefix: String` - key prefix for this backend instance

2. **Configuration:**
   - Endpoint URL (for MinIO, etc.)
   - Region
   - Bucket name
   - Access key / secret key
   - Optional prefix/path style

3. **Key naming:**
   - Format: `{prefix}/{sanitized_namespace}/{path}`
   - Sanitize namespace for S3 key compatibility

4. **Error mapping:**
   - S3 404 → VfsError::NotFound
   - S3 403 → VfsError::PermissionDenied
   - Network errors → VfsError::IoError

5. **Implement VfsBackend trait:**
   - All methods async via rust-s3
   - Streaming for large files (optional optimization)

**Done Criteria:**
- S3Backend implements VfsBackend
- Unit tests with mock or test container (3+ tests)
- Error codes match Node.js semantics
- Credentials configurable

**Type:** auto
**TDD:** true

Create `src/vfs/factory.rs` for backend instantiation:

   - Method to create backend from config
   - Returns `Arc<dyn VfsBackend>`

2. **Factory logic:**
   ```rust
   pub fn create_backend(
       backend_type: VfsBackendType,
       disk_config: Option<&VfsDiskConfig>,
       s3_config: Option<&VfsS3Config>,
   ```

3. **Implementation:**
   - Memory: Return `Arc::new(MemoryBackend::default())`
   - Disk: Validate path, create directory if needed, return DiskBackend
   - S3: Validate credentials, test connection, return S3Backend

4. **Error handling:**
   - Config missing for backend type → clear error
   - Connection failure → VfsError::IoError with details

**Done Criteria:**
- Factory creates all backend types
- Proper error messages for misconfiguration
- Unit tests for factory (3+ tests)
