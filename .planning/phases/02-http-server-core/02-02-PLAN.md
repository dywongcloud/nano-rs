<objective>
Implement virtual host routing that directs HTTP requests to different handlers based on the Host header. Support exact hostname matching with a fallback default handler. This enables multi-tenancy at the routing layer, a core NANO capability.

Purpose: Enable NANO to route requests to different applications based on hostname, supporting multiple isolated apps on one server.
Output: Working virtual host routing with configurable routes and fallback handling.
</objective>

## Decisions from CONTEXT.md

**D-03:** Exact hostname match only (no wildcards or regex patterns for v1)
- Use HashMap<String, RouteTarget> with lowercase keys
- Route lookup: `host.to_lowercase()` as map key

**D-04:** Fallback to default/catch-all handler when no hostname matches
- Return default handler from router if no exact match
- Never return 404 at routing layer; default handler decides response

**D-11:** JSON error response format: `{"error": "...", "message": "...", "code": N}`
- Use this format for routing errors (malformed Host header, etc.)

**D-12:** JS execution failures: generic 500 to client, detailed error in server logs
- In this phase: placeholder handlers that return "OK"
- Actual JS execution comes in Phase 3

   pub async fn virtual_host_handler(
       State(state): State<Arc<AppState>>,
       Host(host): Host,
       request: Request<Body>,
   ) -> impl IntoResponse {
       tracing::debug!("Request received for host: {}", host);
       
       let target = state.router.resolve(&host);
       
       match &target.handler_type {
           HandlerType::StaticResponse(response) => {
               Response::builder()
                   .status(StatusCode::OK)
                   .header("content-type", "text/plain")
                   .body(Body::from(response.clone()))
                   .unwrap()
           }
           HandlerType::JavaScriptEntrypoint(_) => {
               // Phase 3: Will execute JS handler

                   .status(StatusCode::OK)
                   .header("content-type", "text/plain")
                   .body(Body::from("JS handler (Phase 3)"))
                   .unwrap()
           }
       }
   }

use axum::{

async fn setup_test_router() -> Arc<AppState> {
    let default = RouteTarget {
        hostname: "default".to_string(),
        handler_type: HandlerType::StaticResponse("default-handler".to_string()),
    };
    
    let mut router = VirtualHostRouter::new(default);
    
    // Register test routes
    router.register(
        "api.test.com".to_string(),
        RouteTarget {
            hostname: "api.test.com".to_string(),
            handler_type: HandlerType::StaticResponse("api-handler".to_string()),
        },
    );

        "blog.test.com".to_string(),
        RouteTarget {
            hostname: "blog.test.com".to_string(),

#[tokio::test]
async fn test_routes_by_host_header() {
