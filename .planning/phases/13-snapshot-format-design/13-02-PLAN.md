# Phase 13 Plan 02: VFS State Serialization

## Objective

Implement VFS state serialization for inclusion in sliver snapshots — walking the VFS and creating proper tar entries while preserving directory structure and handling binary files correctly.

## Success Criteria

1. VFS can be serialized into tar entries
2. Directory structure is preserved correctly
3. Binary files are handled without corruption
4. VFS state roundtrip tests pass
5. Tar structure validation tests pass

## Context

Building on the sliver format core (Plan 13-01), this plan focuses specifically on VFS serialization. The VFS module from Phases 10-12 provides the storage interface; now we need to serialize that state into the tar format.

## Tasks

### Task 1: Extend VFS types for serialization

**Action:** Add serialization support to existing VFS types

**Add to vfs/mod.rs or vfs/types.rs:**
- `Serialize` trait implementation for VfsEntry
- Helper to walk entire VFS tree
- Methods to convert entries to tar-compatible format

**Commit:** feat(13-02): Add VFS serialization traits

### Task 2: Implement VFS tree walker

**Action:** Create function to walk VFS and collect all entries

```rust
pub fn walk_vfs(root: &VfsRoot) -> Vec<(PathBuf, VfsEntry)>
```

**Should visit:**
- All directories (create parent paths)
- All files (with content)
- Preserve relative paths within VFS root

**Commit:** feat(13-02): Implement VFS tree walker

### Task 3: Create tar entry builder

**Action:** Build tar entries from VFS entries

**Handle file types:**
- Regular files → tar::EntryType::Regular
- Directories → tar::EntryType::Directory
- (Symlinks not supported in v1.1)

**Preserve:**
- File content (binary-safe)
- File permissions (default 0o644 for files, 0o755 for dirs)
- Modification time (optional, use created_at if available)

**Commit:** feat(13-02): Build tar entries from VFS

### Task 4: Extract VFS from tar entries

**Action:** Parse tar entries and reconstruct VFS state

```rust
pub fn extract_vfs(entries: &[tar::Entry]) -> Result<VfsRoot, SliverError>
```

**Handle:**
- Directory entries (create paths)
- File entries (store content)
- Nested paths (create parent directories)
- Ignore non-vfs/ entries

**Commit:** feat(13-02): Extract VFS from tar entries

### Task 5: Binary file handling tests

**Action:** Test binary file preservation

**Test data:**
- PNG image bytes
- UTF-8 text with emoji
- Mixed binary content
- Empty files
- Large files (>1MB)

**Verify:**
- Byte-perfect roundtrip
- No encoding corruption
- Size preserved exactly

**Commit:** test(13-02): Binary file roundtrip tests

### Task 6: Directory structure preservation tests

**Action:** Test complex directory trees

**Test structures:**
```
/data/
  config.json
  nested/
    deep/
      file.txt
/assets/
  logo.png
  fonts/
    font.woff2
```

**Verify:**
- All directories recreated
- Files in correct locations
- Empty directories preserved
- Deep nesting supported

**Commit:** test(13-02): Directory structure tests

### Task 7: Tar structure validation

**Action:** Create validation tests for tar format

**Validate:**
- Tar can be read by standard `tar` command
- Entry names are valid UTF-8
- No duplicate entries
- Required files present (meta.json, heap.bin)
- VFS entries have vfs/ prefix

**Test with external tar:**
```bash
tar -tf test.sliver
```

**Commit:** test(13-02): Tar structure validation tests

### Task 8: Integration with sliver packer

**Action:** Wire VFS serialization into packer

**Modify packer.rs:**
- Accept VfsRoot reference
- Call VFS walker during packing
- Add vfs/ prefixed entries to archive

**Commit:** feat(13-02): Integrate VFS into packer

### Task 9: Integration with sliver unpacker

**Action:** Wire VFS extraction into unpacker

**Modify unpacker.rs:**
- Extract vfs/ entries
- Reconstruct VfsRoot
- Include in UnpackedSliver result

**Commit:** feat(13-02): Integrate VFS into unpacker

### Task 10: End-to-end sliver roundtrip test

**Action:** Complete integration test

**Test scenario:**
1. Create VFS with files and directories
2. Create metadata
3. Create fake heap data (opaque blob)
4. Pack into sliver
5. Unpack from sliver
6. Verify all contents match

**Commit:** test(13-02): End-to-end sliver roundtrip test
