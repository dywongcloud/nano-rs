# Phase 13 Plan 01: Sliver Format Core

## Objective

Implement the core sliver format — a tar-based archive structure for JavaScript isolate snapshots that includes V8 heap state, VFS contents, and metadata.

## Success Criteria

1. SliverMetadata struct defined with JSON serialization
2. Tar packing/unpacking functions work correctly
3. Sliver format specification documented
4. Roundtrip tests pass (pack → unpack → verify)

## Context

Phase 10-12 established the VFS module with in-memory storage and JavaScript bindings. Now we need to define the snapshot format that captures both V8 heap state and VFS contents for migration and fast cold starts.

Design principles:
- **Simple:** Just a tar file, inspectable with `tar -tf`
- **Opaque:** heap.bin is opaque blob, don't parse it
- **Portable:** No host-specific paths or IDs
- **Evolvable:** Format allows future extensions

### Task 5: Implement tar packing

**Action:** Create packer that builds tar archive

**Inputs:**
- heap_data: Vec<u8> (opaque V8 heap blob)
- vfs_root: &VfsRoot (VFS state)

**Output:** Vec<u8> (tar archive bytes)

**Structure:**
```
archive.tar
├── meta.json
├── heap.bin
├── manifest.txt
└── vfs/
    └── [VFS entries as tar entries]
```
