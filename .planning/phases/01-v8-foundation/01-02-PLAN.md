## Critical Pitfall Context (from PITFALLS.md)

### EPT SIGSEGV Bug (Pitfall 1)
- **Root cause:** ArrayBuffer allocations in EPT segments get unmapped by background GC
- **Fix:** Strong v8::Global<Value> sentinel per isolate (D-02)
- **Implementation:** Create sentinel at isolate init, hold until disposal
- **Reference:** This is the AP-02 fix from the Zig version

### HandleScope Pattern (Pitfall 2)
- **Pattern:** Nested HandleScopes for temporary operations
- **Anti-pattern:** Long-lived HandleScopes cause unbounded memory growth
- **Implementation:** Short-lived scopes around V8 operations

### Thread Safety (Pitfall 3)
- **Rule:** Isolates must never move between threads (!Send)
- **Pattern:** Use std::sync::Once for platform initialization
- **Reference:** rusty_v8 issue #1467
</context>

    isolate: v8::OwnedIsolate,
    sentinel: v8::Global<v8::Value>, // Keeps EPT segment mapped

        let isolate = v8::Isolate::new(Default::default());
        let mut isolate = scopeguard::guard(isolate, |i| { /* cleanup */ });
        
        let scope = &mut v8::HandleScope::new(&mut *isolate);
        let sentinel = v8::Global::new(scope, v8::undefined(scope));
        
        Ok(Self {
            isolate: scopeguard::ScopeGuard::into_inner(isolate),
            sentinel,
        })
    }
}
