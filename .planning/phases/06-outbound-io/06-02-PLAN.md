    - Test 3: writer.write(chunk) writes Uint8Array and returns Promise<void>
    - Test 4: Backpressure: write() doesn't resolve until buffer has space
    - Test 5: writer.close() signals end of stream
    - Test 6: writer.abort() signals error and rejects pending writes
    - Test 7: write() with non-Uint8Array throws TypeError
  </behavior>
  <action>
    Extend src/runtime/stream.rs with WritableStream:
    
    WritableStream struct:
    - `underlying_sink` trait object for Rust-side data consumption
    - Internal state: writable, closed, aborted, error
    - HighWaterMark for backpressure (default 1 chunk)
    
    WritableStreamDefaultWriter:
    - `write(chunk: Uint8Array) -> Promise<void>` - writes chunk, resolves when accepted
    - `close() -> Promise<void>` - signals end, resolves when sink processes
    - `abort(reason) -> Promise<void>` - aborts stream with error
    - `ready` getter - Promise that resolves when backpressure clears
    - `desiredSize` getter - number of chunks that can be written without blocking
    
    UnderlyingSink trait:
    ```rust
    pub trait UnderlyingSink {
        fn start(&mut self) -> Result<(), Error>;
        fn write(&mut self, chunk: Bytes) -> impl Future<Output = Result<(), Error>>;
        fn close(&mut self) -> impl Future<Output = Result<(), Error>>;
        fn abort(&mut self, reason: Option<String>) -> impl Future<Output = Result<(), Error>>;
    }
    ```
    
    V8 bindings:
    - `bind_writable_stream(scope, context)` - registers WritableStream constructor
    - Constructor accepts optional {start, write, close, abort} object
    - Bind methods on WritableStream prototype: getWriter
    - Bind methods on WritableStreamDefaultWriter prototype: write, close, abort
    
    Backpressure implementation:
    - Use tokio::sync::mpsc::channel with bounded capacity
    - Writer.write() sends chunk to channel, awaits consumer
    - Consumer (hyper body) pulls from channel
    - When channel is full, write() blocks (yielding) until space available
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS WritableStream → Rust channel | Untrusted JS writes to stream |
| Rust channel → Hyper body | Data flows to network |
| Network → External server | Upload leaves the runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | DoS | Upload size | mitigate | Max request body size limit (100MB default), configurable per-app |
| T-06-09 | DoS | Upload duration | mitigate | Request timeout covers entire upload (30s default) |
| T-06-10 | Information Disclosure | Upload content | accept | User data flows through runtime — standard expectation |
| T-06-11 | DoS | Channel overflow | mitigate | Bounded channel (4 chunks) ensures backpressure propagates |
| T-06-12 | Elevation | Stream reuse | mitigate | WritableStream becomes "locked" when getWriter() called, can't get another writer |
| T-06-13 | DoS | Resource exhaustion | mitigate | Max concurrent streaming uploads per isolate (10), reject with 503 when exceeded |
</threat_model>

    start(controller) {
      this.controller = controller;
    },
    write(chunk) {
      // In real use, this would come from a file or other source
      return Promise.resolve();
    }

  const writer = stream.getWriter();
  
  // Write some data
  const encoder = new TextEncoder();
  await writer.write(encoder.encode('Hello, '));
  await writer.write(encoder.encode('streaming '));
  await writer.write(encoder.encode('world!'));
  await writer.close();
  
  // Send the request
  const response = await fetch('https://httpbin.org/post', {
    method: 'POST',
    body: stream,
    headers: {'Content-Type': 'text/plain'}
  });
  
  const result = await response.json();
  console.log('Server received:', result.data);
  
  return new Response('Upload complete');
}
