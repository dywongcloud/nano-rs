## Bug Fixes During Phase

### Response Headers Extraction Fix
**Issue:** CORS middleware using `response.headers.set()` wasn't persisting headers that could be extracted by `extract_js_response()`.

**Root Cause:** Response constructor stored initial headers directly on the headers object, but `set()` method stored updates in `__headers__`. The extraction code prioritized `__headers__`, missing initial headers.

**Fix:** Response constructor now initializes `__headers__` internal object and stores all headers there, ensuring consistent storage and extraction.

**Files Modified:**
- `src/runtime/apis.rs` — Initialize `__headers__` in Response constructor
- Store initial headers in both locations for compatibility
