# Phase 21.1 Plan 03: Sliver Creation from Directory (No Running App Required) Summary

## Overview

This plan enables sliver creation directly from a directory without requiring the app to be running first. The sliver packs all app files and is runnable standalone. This is particularly useful for static site generators like Astro and Next.js that export to a `dist` directory.

## Changes Made

### 1. CLI Arguments (`src/cli/sliver.rs`)

- Changed `hostname` from `String` to `Option<String>` to make it optional when using `--from-dir`
- Added `--from-dir` option to specify source directory for sliver creation
- Updated all CLI tests to handle the new `Option<String>` type for hostname
- Added tests for `--from-dir` functionality

### 2. Packager Module (`src/sliver/packager.rs`)

Created new module with:

- **`create_sliver_from_directory()`**: Main function to create sliver from directory
  - Validates source directory exists
  - Auto-detects entrypoint (index.js ‚Üí index.html fallback)
  - Creates metadata with custom fields for sliver type and entrypoint
  - Creates placeholder heap for directory-based slivers
  - Loads all files into VFS entries
  - Creates the sliver archive

- **`create_placeholder_heap()`**: Creates a magic header for directory slivers
  - Format: "NANO-DIR-v1\0" + entrypoint path
  - Allows runtime to detect this is a directory-based sliver

- **`detect_entrypoint()`**: Auto-detects entrypoint from directory
  - Priority: index.js ‚Üí index.mjs ‚Üí main.js ‚Üí worker.js ‚Üí index.html
  - Defaults to index.html if no recognized entrypoint found

- **`load_directory_files()`**: Recursively loads all files from directory
  - Preserves directory structure in VFS paths
  - Skips hidden files (starting with .) and .sliver files
  - Converts paths to use forward slashes for cross-platform compatibility

### 3. Main.rs Updates (`src/main.rs`)

- Added import for `create_sliver_from_directory` from packager module
- Modified `handle_sliver_command` to handle `--from-dir` option:
  - If `--from-dir` is provided, use packager to create sliver from directory
  - `--name` is required when using `--from-dir`
  - Hostname defaults to name if not specified
  - Falls back to original hostname-based creation if `--from-dir` not provided
- Fixed type mismatches due to `hostname` becoming `Option<String>`

### 4. Module Export (`src/sliver/mod.rs`)

- Added `pub mod packager;` to export the packager module

## Notes

- The sliver created from directory uses a placeholder heap (not a real V8 snapshot) since the app wasn't running when created. The runtime will detect this via the "NANO-DIR-v1" magic header and create a fresh isolate when the sliver is run.
- Entrypoint detection follows the priority: index.js ‚Üí index.mjs ‚Üí main.js ‚Üí worker.js ‚Üí index.html
- All files in the directory are included in the sliver except hidden files (starting with .) and .sliver files
- The source directory path is stored in metadata for traceability
