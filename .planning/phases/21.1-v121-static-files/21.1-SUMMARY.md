## Issues Fixed During Phase

### Response Headers Extraction Bug
**Discovered:** During Wave 4 testing, Hono.js CORS middleware tests failed

**Error:** Headers set via `response.headers.set()` weren't being extracted properly

**Root Cause:** Response constructor stored initial headers directly on the headers object, but `set()` method stored them in `__headers__`. The extraction code (`extract_js_response`) prioritized `__headers__`, missing headers set at construction time.

**Fix:** Modified Response constructor to:
1. Initialize `__headers__` internal object
2. Store initial headers in `__headers__` (not just on the object directly)
3. Ensure consistency between direct storage and internal storage

**Commit:** `51c1d4bf`: fix(21.1): fix Response headers extraction for CORS middleware

## Usage Examples

### Serve Static HTML Site
```bash
# Config with HTML entrypoint
echo '{"apps":[{"hostname":"site.local","entrypoint":"./dist/index.html"}]}' > config.json
nano-rs run --config config.json

# Or serve entire directory
echo '{"apps":[{"hostname":"site.local","entrypoint":"./dist"}]}' > config.json
nano-rs run --config config.json
```

### Create Sliver from Astro Build
```bash
# Build Astro static site
cd my-astro-blog
npm run build

# Create sliver from dist
nano-rs sliver create --from-dir ./dist --name myblog --tag v1.0

# Run sliver standalone
nano-rs run --sliver myblog.sliver --port 8080
```

### Sliver from Any Directory
```bash
# Create sliver from any directory
nano-rs sliver create --from-dir ./my-static-site --name mysite

# Verify contents
tar -tzf mysite.sliver
# Output: files/, files/index.html, files/style.css, sliver.json
```
