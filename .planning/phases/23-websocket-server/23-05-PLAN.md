       - Get global from context
       - Enter ContextScope
       - Create FunctionTemplate for websocket_pair_constructor
       - Get function from template, set on global as "WebSocketPair"

    3. Implement websocket_pair_constructor FunctionCallback:
       - Create two V8 Objects: server_socket and client_socket using create_websocket_object helper
       - Store server_socket as Global in WS_SERVER_SOCKET thread-local (from tenant_pool.rs)
       - Create pair Object, set keys "0" and "1" (string keys for Object.values() insertion order per D-04)
       - Set key "0" = client_socket, key "1" = server_socket (CF convention: 0=client, 1=server)
       - Return pair via retval.set()

    4. Implement fn create_websocket_object(scope) -> v8::Local&lt;v8::Object&gt;:
       - Create v8::Object
       - Attach methods via v8::Function::new for each callback:
         - "accept" -> ws_accept_callback
         - "send" -> ws_send_callback
         - "close" -> ws_close_callback
         - "addEventListener" -> ws_add_event_listener_callback
       - Set "readyState" property to v8::Integer 0 (CONNECTING)
       - Set "binaryType" property to v8::String "arraybuffer"
       - Return the object

    5. Implement ws_accept_callback FunctionCallback:
       - Set WS_ACCEPTED thread-local to true
       - Call set_ws_readystate(scope, 1) to set OPEN
       - No return value

    6. Implement ws_send_callback FunctionCallback:
       - Check WS_ACCEPTED thread-local. If false, throw TypeError("WebSocket is not accepted") per D-14b and return.
       - Read args.get(0). If it's a v8::String, convert to Rust String and create tungstenite::Message::Text. If it's an ArrayBuffer or ArrayBufferView, extract bytes and create tungstenite::Message::Binary.
       - Send the message via WS_OUTBOUND thread-local. If WS_OUTBOUND is None or send fails, silently ignore (connection already closing).

    7. Implement ws_close_callback FunctionCallback:
       - Read optional code from args.get(0) (default 1000) and reason from args.get(1) (default "")
       - Set readyState to 2 (CLOSING) via set_ws_readystate
       - Create tungstenite::Message::Close with CloseFrame containing code and reason
       - Send via WS_OUTBOUND thread-local

    8. Implement ws_add_event_listener_callback FunctionCallback:
       - Read event type from args.get(0) as string
       - Read handler from args.get(1), verify is_function()
       - Create v8::Global from the function
       - Match event type:
         - "message" -> push to WS_MESSAGE_HANDLERS thread-local
         - "close" -> push to WS_CLOSE_HANDLERS thread-local
         - "error" -> push to WS_ERROR_HANDLERS thread-local
         - anything else -> ignore silently

    9. Add binaryType accessor enforcement per D-18b. When creating the socket object in create_websocket_object, use v8::ObjectTemplate with an accessor (getter returns "arraybuffer", setter throws TypeError("binaryType is read-only: only arraybuffer supported")). Alternatively, if ObjectTemplate is complex, set a regular property and add a separate "Object.defineProperty" call to make it non-writable/non-configurable. The simplest approach: use a simple property set (already done in step 4) and add a setter FunctionCallback that throws TypeError. Store the binaryType as a regular property; the setter on the prototype intercepts writes.

    10. In src/runtime/apis.rs, add a new method to RuntimeAPIs impl block:
