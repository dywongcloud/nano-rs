    Reload sequence (async method):
    1. Validate new_config (return Err if invalid)
    2. Compute ConfigDiff
    3. Create new AppRegistry from new_config
    4. Start new worker pools for "added" apps

    9. Update existing pools for "updated" apps (or stop/start)

    - ValidationError: new config is invalid
    - DrainTimeoutError: drain exceeded 30 second limit
    - PoolStartError: failed to start new worker pool

      ```rust
      pub async fn run_runtime(config_path: PathBuf) -> Result<()> {
          // Load initial config
          let config = load_config(&config_path).await?;
          let registry = Arc::new(RwLock::new(AppRegistry::from_config(&config)?));
          let work_queue = Arc::new(Mutex::new(WorkQueue::new(4)));
          let request_drain = Arc::new(RequestDrain::new());
          
          // Create hot reloader
          let hot_reloader = Arc::new(HotReloader::new(
              registry.clone(),
              work_queue.clone(),
              request_drain.clone()
          ));
          
          // Start config watcher task
          let watcher_path = config_path.clone();
          let watcher_reloader = hot_reloader.clone();
          tokio::spawn(async move {
              let mut watcher = ConfigWatcher::new(watcher_path);
              loop {
                  match watcher.watch().await {
                      ConfigEvent::Modified { .. } => {
                          match load_config(&config_path) {
                              Ok(new_config) => {
                                  if let Err(e) = watcher_reloader.reload(new_config).await {
                                      error!("Hot reload failed: {}", e);
                                  }
                              }
                              Err(e) => error!("Failed to load new config: {}", e),
                          }
                      }
                      _ => {}
                  }
              }
          });
          
          // Start HTTP server with registry
          let app_state = AppState::new(registry, work_queue, request_drain, hot_reloader);
          start_server(app_state).await
      }
      ```
    
    Add CLI flag parsing for --config path (default: nano.json).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| file_system → ConfigWatcher | Untrusted file changes trigger reloads |
| old_config → new_config | Config swap could expose apps |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | DoS | Rapid config changes | mitigate | Debounce changes (2s minimum between reloads), max 1 reload per 5 seconds |
| T-05-12 | Tampering | Malicious config file | mitigate | Validate new config before applying; reject if invalid, keep old config |
| T-05-13 | Elevation | Race during reload | mitigate | Atomic Arc::swap for registry update; drain ensures no in-flight requests on old config |
| T-05-14 | DoS | Drain timeout abuse | mitigate | 30 second drain timeout max; force swap even if requests pending (they get 503) |
| T-05-15 | Information Disclosure | Config file path | accept | Config path is argument, not secret; no sensitive data in requirements |
</threat_model>
