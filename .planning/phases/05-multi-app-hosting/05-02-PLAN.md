<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS code → Memory Limiter | Untrusted JS could try to exhaust memory |
| JS code → Timeout Watchdog | Untrusted JS could try to run indefinitely |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | DoS | Memory limit bypass | mitigate | Use V8's built-in heap limits (not just external tracking), enable OOM callback |
| T-05-07 | DoS | Timeout bypass | mitigate | Use v8::Isolate::terminate_execution() which forcefully stops JS, plus async timeout at Rust level |
| T-05-08 | Tampering | Limit configuration | mitigate | Validate limits in config: min 16MB memory, max 2GB; min 1s timeout, max 5min |
| T-05-09 | Information Disclosure | Heap stats | accept | Heap statistics don't expose sensitive data; acceptable for debugging |
| T-05-10 | Availability | OOM/Timeout cleanup | mitigate | Ensure isolate cleanup runs even when terminated; use Drop guards |
</threat_model>
