<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config_file → loader | Untrusted file content crosses into runtime |
| env_vars → JS global | Host environment leaks into isolate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Config loader | mitigate | Validate JSON schema, reject unknown fields with #[serde(deny_unknown_fields)] |
| T-05-02 | Information Disclosure | Environment variables | mitigate | Only inject explicitly configured env vars, not entire host environment |
| T-05-03 | DoS | Config validation | mitigate | Enforce limits: max 1000 apps, max 10 env vars per app, max 1MB config file size |
| T-05-04 | Elevation | Entrypoint path | mitigate | Validate entrypoint is within allowed base directory (prevent ../../../etc/passwd) |
| T-05-05 | Repudiation | Config loading | accept | Log config load events with timestamp and source file hash |
</threat_model>
