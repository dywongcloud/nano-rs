    # WebAssembly Async Event Loop Implementation
    
    **Phase:** 28 — WASM Async Event Loop  
    **Date:** 2026-05-03  
    **Status:** ✅ Complete
    
    ## Overview
    
    This document describes the implementation of async Promise resolution for WebAssembly
    operations in NANO. Before Phase 28, WASM operations like `WebAssembly.compile()` and
    `WebAssembly.instantiate()` would return "Promise still pending" errors because V8's
    microtask checkpoint was only called once, not in a loop until the Promise resolved.
    
    ### The Problem
    
    WebAssembly operations in JavaScript return Promises:
    
    ```javascript
    const module = await WebAssembly.compile(wasmBytes);  // Returns Promise
    const instance = await WebAssembly.instantiate(module, {});  // Returns Promise
    ```
    
    V8 requires microtask checkpoints to advance Promise resolution. The original code:
    
    ```rust
    scope.perform_microtask_checkpoint();  // Only ONE checkpoint
    match promise.state() {
        v8::PromiseState::Pending => {
            return Err(anyhow!("Promise still pending"));  // ❌ FAILS
        }
        ...
    }
    ```
    
    A single checkpoint is insufficient for async operations that need to wait for
    I/O or background compilation.
    
    ### The Solution
    
    Create an async Promise resolver that loops with timeout:
    
    ```rust
    pub fn resolve_promise_async(
        scope: &mut ContextScope<HandleScope>,
        promise: Local<Promise>,
        timeout: Duration,
    ) -> Result<Local<Value>> {
        let start = Instant::now();
        
        while promise.state() == v8::PromiseState::Pending {
            if start.elapsed() > timeout {
                return Err(anyhow!("Promise resolution timeout"));
            }
            
            // Advance V8 microtasks
            scope.perform_microtask_checkpoint();
            
            // Yield to Tokio runtime for async I/O
            if let Some(handle) = get_worker_runtime() {
                handle.block_on(tokio::task::yield_now());
            }
            
            // Cooperate with other threads
            std::thread::yield_now();
        }
        
        // Promise is now resolved
        match promise.state() {
            v8::PromiseState::Fulfilled => Ok(promise.result(scope)),
            v8::PromiseState::Rejected => Err(...),
            _ => unreachable!(),
        }
    }
    ```
    
    ## Architecture
    
    ### V8 Microtask Integration
    
    V8 uses microtasks for Promise resolution. Each checkpoint processes:
    - Promise callbacks (`.then()`, `.catch()`)
    - Async/await continuations
    - Microtask queue items
    
    Multiple checkpoints are required because:
    1. WASM compilation may need multiple async steps
    2. Each step schedules new microtasks
    3. Microtask checkpoint only processes one "batch"
    
    ### Tokio Runtime Bridge
    
    NANO worker threads have thread-local Tokio runtime handles:
    
    ```rust
    thread_local! {
        static WORKER_RUNTIME: RefCell<Option<tokio::runtime::Handle>> = ...;
    }
    ```
    
    The Promise resolver uses `block_on(yield_now())` to allow other Tokio tasks
    (like file system reads via `Nano.fs.readFile()`) to make progress.
    
    ## Implementation
    
    ### Location: src/wasm/async_support.rs
    
    The core implementation provides:
    
    - `AsyncPromiseResolver` struct with configuration
    - `resolve_promise_async()` main entry point
    - `run_event_loop_tick()` helper for single iteration
    - Timeout handling with deadline checking
    - Integration with thread-local WORKER_RUNTIME
    
    ### Timeout Configuration
    
    Default timeout: 30 seconds
    - Configurable via parameter
    - Checked each iteration (sub-second granularity)
    - Returns error on timeout (doesn't hang forever)
