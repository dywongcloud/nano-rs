  <name>Task 1: Create COLD_START.md terminology guide</name>
  <files>docs/COLD_START.md</files>
  <action>
Create comprehensive cold start terminology guide that defines and distinguishes four distinct timing metrics:

1. **PROCESS BOOT** (~60ms): Time from binary execution to HTTP server ready. One-time per process start. Includes:
   - V8 platform initialization
   - Configuration loading
   - HTTP server binding
   - Worker pool creation

2. **SLIVER RESTORATION** (~267µs): Time from sliver file to first request handling. Per-isolate from snapshot:
   - Tar archive extraction (if needed)
   - V8 isolate creation from snapshot blob
   - VFS state restoration
   - Context initialization

3. **CONTEXT RESET** (~5ms): Time between requests in same isolate. Per-request:
   - Context cleanup (previous request)
   - Context creation (new request)
   - Global object reset
   - API bindings re-application

4. **FRESH ISOLATE** (~50-100ms): Time to create new isolate without sliver:
   - V8 isolate creation from scratch
   - Heap initialization
   - Context creation
   - JS compilation

Include measurement methodology section explaining:
- How each metric was measured (tools, methodology)
- Test environment (hardware, OS, V8 version)
- Statistical significance (sample size, variance)
- Comparison to other runtimes (Node.js, Deno, Cloudflare Workers)
