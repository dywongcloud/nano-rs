# Plan 21.2-02: Fix Server Process Cleanup on Error/Termination

## Goal
Fix the server process cleanup to properly release TCP ports and terminate worker threads when the server encounters JavaScript errors or receives termination signals.

## Background
When nano-rs encounters a JavaScript syntax error or other fatal app errors, the server process doesn't fully release its port on shutdown, causing subsequent test runs to timeout on the same port.

The error shows:
```
Fatal error: Error: Server on port 8898 did not start within 10000ms
```

The port remains in a zombie state (likely TIME_WAIT or held by orphaned worker threads), preventing new servers from binding to the same port.

## Requirements

### REQ-21.2-02-01: Graceful Shutdown Sequence
Server must implement proper graceful shutdown:
1. Stop accepting new connections
2. Close existing connections (with timeout)
3. Terminate all worker threads
4. Close the TCP socket
5. Allow immediate port reuse

### REQ-21.2-02-02: Error Scenario Cleanup
Cleanup must run even when:
- JavaScript isolate panics
- Syntax errors cause early exit
- Worker threads crash
- Async tasks are still pending

### REQ-21.2-02-03: Socket Reuse
Server should set socket option `SO_REUSEADDR` to allow quick port reuse during development/testing.

### REQ-21.2-02-04: Signal Handling
Properly handle SIGTERM, SIGINT to trigger graceful shutdown even during error scenarios.

## Implementation

### 1. Add Socket Reuse Option
**File:** `src/http/server.rs` (where the TCP socket is created)

Add `SO_REUSEADDR` socket option before binding:

```rust
use std::net::TcpListener;

let listener = TcpListener::bind(addr)?;
listener.set_reuse_address(true)?; // Allow port reuse
```

Or with tokio:
```rust
use tokio::net::TcpSocket;

let socket = TcpSocket::new_v4()?;
socket.set_reuseaddr(true)?;
socket.bind(addr)?;
let listener = socket.listen(128)?;
```

### 2. Implement Graceful Shutdown Handler
**File:** `src/http/server.rs` or `src/worker/pool.rs`

Create a shutdown coordinator:

```rust
pub struct GracefulShutdown {
    shutdown_tx: broadcast::Sender<()>,
    worker_handles: Vec<JoinHandle<()>>,
}

impl GracefulShutdown {
    pub async fn shutdown(self, timeout: Duration) {
        // Signal all workers to stop
        let _ = self.shutdown_tx.send(());
        
        // Wait for workers with timeout
        for handle in self.worker_handles {
            tokio::time::timeout(timeout, handle).await.ok();
        }
    }
}
```

### 3. Ensure Cleanup on Panic/Error
**File:** `src/http/server.rs`

Use `scopeguard` or manual `Drop` implementation to ensure cleanup:

```rust
struct ServerGuard {
    pool: Option<WorkerPool>,
    listener: Option<TcpListener>,
}

impl Drop for ServerGuard {
    fn drop(&mut self) {
        // Force cleanup of worker pool
        if let Some(pool) = self.pool.take() {
            pool.force_shutdown();
        }
    }
}
```

### 4. Fix Worker Pool Termination
**File:** `src/worker/pool.rs`

Ensure worker threads don't keep the process alive:

```rust
impl WorkerPool {
    pub fn force_shutdown(&mut self) {
        // Signal all workers
        self.shutdown.store(true, Ordering::SeqCst);
        
        // Clear the work queue
        while self.queue.try_recv().is_ok() {}
        
        // Workers should check shutdown flag and exit
    }
}
```

## Testing

### Test 1: Port Reuse After Syntax Error
```rust

async fn test_port_reuse_after_error() {
    // Start server with invalid JS
    let result = start_server_with_js("syntax error here").await;
    assert!(result.is_err());
    
    // Immediately start server on same port
    let result = start_server_with_js("export default { fetch: () => {} }").await;
    assert!(result.is_ok());
}
```

### Test 2: Worker Thread Cleanup
```rust
#[tokio::test]
async fn test_workers_terminate_on_error() {
    // Check thread count before
    let threads_before = get_thread_count();
    
    // Start and crash server
    start_and_crash_server().await;
    
    // Check thread count after (with small delay)
    sleep(Duration::from_millis(100)).await;
    let threads_after = get_thread_count();
    
    // Should be back to ~original count
    assert!(threads_after <= threads_before + 2);
}
```
