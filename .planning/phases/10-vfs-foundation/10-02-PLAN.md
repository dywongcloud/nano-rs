## Integration Points

The VFS must integrate with the existing architecture:

1. **NanoIsolate** (src/v8/isolate.rs): Owns the V8 isolate
   - Add VFS field: `vfs: IsolateVfs`
   - Initialize with hostname-derived namespace

2. **WorkerPool** (src/worker/pool.rs): Creates isolates per-app
   - Pool is per-app (WorkerPool per hostname)
   - All workers for same hostname share VFS namespace concept
   - Actually: each worker thread has its own isolate, but VFS needs shared state

3. **ContextManager** (src/worker/context.rs): Manages context reset
   - Context reset (~5ms) should preserve VFS state
   - VFS persists across context resets within same isolate

## Design Decision: Shared vs Per-Isolate VFS

Per STATE.md D-17 and VFS-02: Per-isolate filesystem namespace.

Two implementation options:
1. **Shared backend**: All isolates for same hostname share memory backend
   - Pros: File changes visible across workers immediately
   - Cons: Need Arc<RwLock<>> around backend
2. **Per-isolate backend**: Each isolate has own MemoryBackend
   - Pros: True isolation even within same app (workers isolated)
   - Cons: Files written in one worker not visible in others

Decision: Shared backend with namespace key prefix:
- Backend storage key: "{hostname}::{normalized_path}"
- Each isolate accesses same backend but with its namespace prefix
- This matches the mental model: "my app's files"

   ```rust
   #[derive(Debug, Clone, PartialEq, Eq, Hash)]
   pub struct VfsNamespace(String);
   
   impl VfsNamespace {
       /// Create namespace from app hostname
       pub fn from_hostname(hostname: &str) -> Self {
           // Sanitize hostname: lowercase, replace dots with underscores
           let sanitized = hostname.to_lowercase()
               .replace('.', "_")
               .replace('-', "_");
           Self(sanitized)
       }
       
       pub fn as_str(&self) -> &str { &self.0 }
   }
   ```

2. Define IsolateVfs struct:
   ```rust
   pub struct IsolateVfs {
       namespace: VfsNamespace,
       backend: Arc<dyn VfsBackend>, // shared across isolates
   }
   ```

3. Implement IsolateVfs:
   - new(namespace: VfsNamespace, backend: Arc<dyn VfsBackend>) -> Self
   - namespace() -> &VfsNamespace: Getter
   - read(path: &str) -> VfsResult<Vec<u8>>: 
     * Normalize path
     * Prefix with namespace
     * Call backend.read()
   - write(path: &str, content: &[u8]) -> VfsResult<()>
   - exists(path: &str) -> VfsResult<bool>
   - delete(path: &str) -> VfsResult<()>
   
4. Key implementation details:
   - Path prefixing: "{namespace}::{normalized_path}"
   - Example: hostname "api.example.com", path "/data/config.json"

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App A ↔ App B | Different hostnames must never share VFS namespace |
| Worker ↔ VFS | Same-app workers share backend, different apps isolated |
| JS runtime → VFS | JS will call VFS via IsolateVfs (Phase 12) |
