## Design Context

The VFS is the foundation for container-image semantics in NANO. It must:
- Support per-isolate file namespaces (Phase 10 Plan 02)
- Be serializable for snapshots (Phase 13)
- Support pluggable backends (Phase 11: disk/S3)
- Expose a JS API (Phase 12)

### Trait Design
Use trait-based architecture for backend abstraction:
- `VfsBackend`: Core operations (read, write, exists, delete)
- `FileSystem`: User-facing API that delegates to backend

### In-Memory Storage
Use `DashMap<String, Vec<u8>>` for concurrent, fast access:
- Keys: normalized file paths
- Values: file contents (bytes)
- Thread-safe: DashMap handles concurrent reads/writes
</context>
