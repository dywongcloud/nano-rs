3. Implement WorkerPool::new(hostname: String, worker_count: usize) -> Self:
   - Spawn N worker threads
   - Each thread creates its own NanoIsolate (thread-local ownership)
   - Each thread runs an event loop receiving HandlerTask via MPSC channel

4. Worker thread loop (per thread):
   ```rust
   thread::spawn(move || {
       // Initialize V8 isolate (thread-local - never moves)
       let mut isolate = NanoIsolate::new().expect("Failed to create isolate");
       
       // Event loop
       while let Ok(task) = task_rx.recv() {
           // Create HandlerContext
           let context = HandlerContext {
               entrypoint: task.entrypoint,
               request: task.request,
           };
           
           // Execute handler (blocking call - use pollster)
           let result = pollster::block_on(
               execute_handler(&mut isolate, context)
           );
           
           // Send response back
           let _ = task.response_tx.send(result);
       }
   })
   ```

5. Implement WorkerPool::dispatch(&self, task: HandlerTask) -> Result<()>
   - For now: round-robin to workers (affine dispatch comes in Plan 02)
   - Send task via MPSC channel

6. Implement WorkerPool::shutdown(self) -> Result<()>
   - Drop all task_tx (signals workers to exit)
   - Join all threads

Key constraints from POOL-05:
- Each isolate is created IN the worker thread
- Isolate never leaves the thread (thread-local ownership)
- Use pollster::block_on for async handler execution in sync thread

| Worker thread boundary | Each worker thread owns its isolate - no cross-thread isolate access |
| MPSC channel | Tasks cross from async runtime to sync worker threads |
