- [x] **HTTP-02**: Virtual host routing via Host header (exact hostname match)
- [x] **HTTP-03**: Request/Response object mapping (WinterCG compatible)
- [x] **HTTP-04**: Headers API implementation
- [x] **HTTP-05**: URL/URLSearchParams implementation

### JavaScript Runtime APIs ✅

- [x] **API-01**: fetch() handler interface (Cloudflare Workers style)
- [x] **API-02**: console.log/warn/error output to structured logs
- [x] **API-03**: TextEncoder/TextDecoder for UTF-8
- [x] **API-04**: setTimeout/setInterval/clearTimeout/clearInterval (event loop integration)

### WorkerPool & Dispatch ✅

- [x] **POOL-01**: WorkerPool per virtual host (N worker threads)
- [x] **POOL-02**: WorkQueue: bounded MPSC channel (256-slot capacity)
- [x] **POOL-03**: Affine dispatch: hostname → pool index → worker thread
- [x] **POOL-04**: Context reset: dispose/recreate V8 context between requests (~5ms)
- [x] **POOL-05**: Isolate-per-thread enforcement (never move isolates across threads)

### Multi-App Hosting ✅

- [x] **HOST-01**: JSON config file for app definitions (hostname → entry point)
- [x] **HOST-02**: Per-app memory limits enforced
- [x] **HOST-03**: Per-app timeout enforcement (watchdog timer)
- [x] **HOST-04**: Per-app environment variables injection
- [x] **HOST-05**: Hot-reload via config file watcher (2s poll interval)
- [x] **HOST-06**: Graceful drain: complete in-flight requests before config swap

### Outbound I/O ✅

- [x] **IO-01**: Outbound fetch() via tokio (non-blocking)
- [x] **IO-02**: ReadableStream for response body streaming
- [x] **IO-03**: WritableStream for request body streaming

### Production Features ✅

- [x] **PROD-01**: Structured JSON logging (ts, level, event, hostname, request_id)
- [x] **PROD-02**: Metrics endpoint (/_admin/metrics): request counts, latency histograms, error rates
- [x] **PROD-03**: Graceful shutdown: SIGTERM/SIGINT handling with request drain
- [x] **PROD-04**: OOM detection and heap limit enforcement
- [x] **PROD-05**: HTTP Admin API on separate port (8889) with API key authentication
- [x] **PROD-06**: Unix domain socket for local admin access (/var/run/nano/control.sock)
- [x] **PROD-07**: Runtime app management: add/remove/disable/scale without restart
- [x] **PROD-08**: Admin diagnostics endpoint (/admin/isolates) with ps-style output

### Framework Compatibility ✅

- [x] **FRAME-01**: Hono.js apps run without modification (test with hello-world + middleware)
- [x] **FRAME-02**: Next.js static export apps serve correctly (HTML/CSS/JS assets)

## Out of Scope (Unchanged)

| Feature | Reason |
|---------|--------|
| npm package resolution | Apps are single-file; bundling is user responsibility |
| TypeScript/JSX transpilation | User must bundle with esbuild/tsc beforehand |
| Native module support (node-gyp) | Breaks isolate security model |
| Subprocess spawning | Security boundary violation |
| Real filesystem access | VFS only; no disk I/O from JS |
| Built-in horizontal clustering | Single-machine focus; use external LB for scale |
| Global edge network | Self-hosted; no CDN integration |
| Durable Objects/stateful coordination | Adds complexity; use external DB |
| HTML Rewriting API | Very high complexity; niche use case |
| TCP/UDP socket outbound | Security risk; use outbound fetch() instead |
| queueMicrotask | WinterTC gap; workaround: Promise.resolve() |
| atob/btoa | WinterTC gap; use Buffer or TextEncoder instead |
