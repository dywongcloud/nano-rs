## Overview

NANO v1.0 is a production-ready multi-tenant JavaScript edge runtime. One OS process hosts many isolated V8 isolates, each running separate JavaScript applications with millisecond cold starts, zero container overhead, and strong per-app isolation.

**Core Value:** Skip the container fleet entirely — one binary, one config file, many isolated apps.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Rust + rusty_v8 over Zig | Pre-built V8, type-safe bindings, stable ecosystem |
| Context reset (not new isolate) | 5ms vs 50-100ms per request |
| ring crate over V8 crypto | Safer, avoids V8 internal complexity |
| No npm/import resolution | Simplifies runtime, keeps isolates lightweight |
| WorkerPool per virtual host | Resource isolation between tenants |
| Promise-based async APIs | WinterTC compliance, consistent interface |

## Architecture Highlights

```
TCP accept (main thread)
    ↓
Virtual host routing → AppRegistry lookup
    ↓
WorkQueue dispatch (bounded MPSC)
    ↓
Worker thread enters V8 isolate
    ↓
Context reset (~5ms)
    ↓
JavaScript fetch() handler executes
    ↓
HTTP response written
    ↓
Context reset for next request
```
