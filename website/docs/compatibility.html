Compatibility

Node.js compatibility, polyfills, and the nano-compat library

Compatibility Overview

NANO is a WinterTC-compatible runtime, not a Node.js runtime. This means it provides web-standard APIs (fetch, Request, Response, etc.) rather than Node.js-specific APIs (fs, path, http, etc.).

For applications that need Node.js compatibility, NANO provides the nano-compat library - a collection of polyfills and compatibility shims.

nano-compat Library

The nano-compat library provides polyfills for common Node.js APIs that are not part of the WinterTC standard.

Installation
npm install @nano-rs/compat
Usage
// Apply all polyfills
import '@nano-rs/compat';

// Or import specific modules
import { Buffer } from '@nano-rs/compat/buffer';
import { path } from '@nano-rs/compat/path';

What is WinterTC?

WinterTC (Web-interoperable Runtimes Community Group) defines standards for JavaScript runtimes that work across different environments. Unlike Node.js which has its own APIs, WinterTC runtimes use web-standard APIs.

Key Principles

  • Web Standards First: Use fetch, Request, Response instead of Node's http module
  • Isomorphic Code: Same code runs in browser, edge, and serverless
  • Minimal Surface: Only standardized APIs, no runtime-specific extensions

WinterTC Compliance

NANO implements the WinterTC standard for edge runtimes.
