Quick Start

Get up and running with NANO in under 5 minutes

1

Install NANO

Download the latest release from GitHub and install.

Install via GitHub Releases
# Download latest release (Linux/macOS)
curl -L https://github.com/nano-rs/nano-rs/releases/latest/download/nano-rs-$(uname -s)-$(uname -m).tar.gz | tar xz

# Move to PATH
sudo mv nano-rs /usr/local/bin/

# Verify installation
nano-rs --version
2

Create Your First App

Create a simple JavaScript app that responds to HTTP requests.

app.js
// Simple HTTP handler using WinterTC fetch API
export default {
  async fetch(request) {
    const url = new URL(request.url);
    
    if (url.pathname === '/') {
      return new Response('Hello from NANO!', {
        headers: { 'content-type': 'text/plain' }
      });
    }
    
    if (url.pathname === '/json') {
      return Response.json({ 
        message: 'Hello from NANO!',
        runtime: 'nano-rs',
        wintertc: true
      });
    }
    
    return new Response('Not Found', { status: 404 });
  }
};
3

Configure and Run

Create a configuration file and start the NANO server.

nano.json - Multi-tenant Configuration
{
  "port": 8080,
  "workers": 4,
  "apps": [
    {
      "name": "my-app",
      "entry": "app.js",
      "domain": "localhost"
    },
    {
      "name": "api-service",
      "entry": "api.js",
      "domain": "api.localhost"
    }
  ]
}

Multi-tenant routing: The Host header determines which app handles the request. curl -H "Host: api.localhost" routes to the api-service app, while curl -H "Host: localhost" routes to my-app.

Run the server
# Start the server
nano-rs run --config nano.json

# In another terminal, test your app
# Note: Use the Host header for multi-tenant routing
curl -H "Host: localhost" http://localhost:8080/
# Output: Hello from NANO!

curl -H "Host: localhost" http://localhost:8080/json
# Output: {"message":"Hello from NANO!","runtime":"nano-rs","wintertc":true}

# The Host header routes to the correct app isolate
# Each app can have its own domain configured in nano.json
4

Create a Sliver

Slivers are portable snapshots that enable sub-millisecond cold starts. Create one from your running app.

Create a sliver
# Create a sliver from the running app
nano-rs sliver create localhost --name my-app --tag v1.0

# The sliver is now stored and can be used for instant cold starts
5

List and Manage Slivers

View all your slivers and deploy them instantly.

Manage slivers
# List all slivers
nano-rs sliver list

# Show detailed info
nano-rs sliver list --verbose

# Deploy from a sliver
nano-rs app deploy --sliver my-app-v1.0 --name my-app
