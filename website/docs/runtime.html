Runtime

Understanding NANO runtime configuration and behavior

Configuration

NANO uses JSON configuration files to define the runtime behavior.

nano.json
{
  "port": 8080,
  "workers": 4,
  "apps": [
    {
      "name": "my-app",
      "entry": "app.js",
      "domain": "app.example.com"
    }
  ],
  "slivers": {
    "enabled": true,
    "cache_size": 100
  }
}

Worker Pool

NANO uses a worker pool to handle incoming requests efficiently.

Configuration

{
  "workers": 4,        // Number of worker threads
  "queue_size": 1000,  // Max queue size for pending requests
  "timeout": 30000     // Request timeout in milliseconds
}

Each worker maintains a thread-local isolate for fast context switching between requests.

Performance Tuning

Memory

Configure heap limits and memory pooling.

{
  "memory": {
    "heap_limit": 128,      // MB per isolate
    "pool_size": 10,        // Pre-warmed isolates
    "gc_threshold": 0.8     // Trigger GC at 80% usage
  }
}

Slivers

Enable and configure sliver snapshots for cold starts.

{
  "slivers": {
    "enabled": true,
    "cache_size": 100,      // Max slivers to cache
    "compression": "lz4",   // Compression algorithm
    "auto_create": true    // Auto-create slivers on deploy
  }
}
